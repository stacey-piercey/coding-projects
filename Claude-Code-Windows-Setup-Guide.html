Claude Code

Windows Setup Guide

For new team members joining coding-projects

Overview

This guide walks you through setting up Claude Code on Windows, connecting it to Git and GitHub, and joining the coding-projects repository. Follow each step in order.

What you will install:

  • Node.js — required to run Claude Code
  • Git — version control software
  • Claude Code — the AI coding agent

Prerequisites

Before starting, make sure you have:

  • A Claude account (claude.ai) — Pro or Max plan required for full Claude Code access
  • A GitHub account (github.com) — free tier is fine
  • Windows 10 version 2004 or later (build 19041+)

Step 1: Install Node.js

  1. Go to nodejs.org and download the LTS version for Windows
  2. Run the installer with default settings
  3. When asked about "automatically install the necessary tools" — check this box. It installs Chocolatey, Python, and Visual Studio Build Tools which you will need for agentic development
  4. A separate PowerShell window will open and run for a few minutes — let it finish completely before moving on
  5. Close ALL open terminals when done

Verify the install by opening a new terminal and typing:

node --version

Step 2: Install Git

  1. Go to git-scm.com and download the Windows installer
  2. Run with default settings. When asked about the default editor, choose VS Code if you have it, otherwise Notepad is fine
  3. Open a new terminal and configure your identity (replace with your actual name and email):
git config --global user.name "Your Name"
git config --global user.email "you@email.com"

Step 3: Install Claude Code

Open a new terminal (search for PowerShell or Command Prompt in the Start menu) and run:

npm install -g @anthropic-ai/claude-code

Then launch Claude Code:

claude

A welcome/onboarding screen will appear — press Enter or Escape to get past it. Then type /login to open a browser window. Log in with your Anthropic account and authorize. Return to the terminal when done.

Step 4: Set Up Your Project Folder

Navigate to your projects drive. If you have a secondary drive (e.g. E:), use it to keep your C: drive lean. Use the /d flag to switch drives in Windows:

cd /d E:\Projects
The /d flag is required when switching between drives on Windows. Without it, cd will not change drives.

Your prompt should now show E:\Projects>. If you are only using C:, run cd C:\Projects instead (create the folder first with mkdir C:\Projects).

Step 5: Clone the Repository

Clone the shared coding-projects repository from GitHub:

git clone https://github.com/stacey-piercey/coding-projects.git
cd coding-projects

A browser window may open asking you to sign in to GitHub — authorize it.

You should see: branch 'main' set up to track 'origin/main' — this confirms everything is connected.

Step 6: Verify Your Setup

Run the Claude Code health check from inside your project folder:

/doctor

This runs a health check and confirms your setup is working correctly. Report any errors to the team lead before proceeding.

Troubleshooting

"npm is not recognized" error

Close your terminal completely and open a fresh one. The Node.js installer updates your system PATH, but terminals that were already open won't see the change until restarted.

"cd" won't switch to another drive

Windows requires the /d flag to switch drives. Use cd /d E:\Projects instead of cd E:\Projects.

"src refspec main does not match any" error on push

This happens when trying to push an empty repository. Git won't create a branch until there is at least one commit. Create an initial commit first:

echo "# project-name" > README.md
git add README.md
git commit -m "first commit"
git push -u origin main

Computer Use is not available in Settings

Computer Use is currently a macOS-only research preview. It is not yet available on Windows. All other Claude Code features work fully on Windows.

Daily Workflow

Once set up, your typical Claude Code session will look like this:

  1. Open a terminal and navigate to your project: cd /d E:\Projects\coding-projects
  2. Pull the latest changes from GitHub: git pull
  3. Start Claude Code: claude
  4. Work with Claude — it handles commits as you go
  5. Push your work to GitHub: git push
Questions? Reach out to the team lead before your first session.
